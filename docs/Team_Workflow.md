# Team Workflow - InterXview Project

---

## 1. Team & Roles
- **Number of Members:** 4
- **Members and Responsibilities:**
  - **Bara'a (براءة)** – Backend Developer, Data Lab, Interview Interface Lead
  - **Duha (ضحى)** – Frontend Developer
  - **Mohammed Zyoud (محمد الزيود)** – Frontend Developer
  - **Hammam (همام)** – Backend & Database Developer

_All team members contributed across multiple features. However, Bara'a was responsible for the main interface integration and overall technical coordination._

---

## 2. Task Management
- **Tool:** Jira
- **Task Distribution:**  
  Tasks are assigned based on each member’s specialization (Frontend, Backend, Data). Collaboration was encouraged, and shared ownership was applied when needed.
- **Workflow Stages:**  
  - Backlog → In Progress → Review → Done
- **Task Tracking:**  
  Jira was used to track progress, priorities, deadlines, and task completion status.

---

## 3. Communication & Meetings
- **Tools:** Microsoft Teams & WhatsApp
- **Meetings:**  
  - Three meetings per week for planning, progress tracking, and issue resolution
- **Documentation:**  
  Key decisions, technical discussions, and updates were documented through Jira and meeting notes.

---

## 4. Technology Stack & Development Tools
- **Main Stack (PERN Stack):**
  - **PostgreSQL** – Relational database for structured data storage
  - **Express.js** – Backend framework for API development
  - **React.js** – Frontend library for building interactive user interfaces
  - **Node.js** – Server-side runtime environment
- **Version Control:**
  - **Git & GitHub** were used for source code management, collaboration, and version tracking
  - Feature branches and pull requests ensured clean and controlled development
- **Presentation Tool:**
  - **Prezi** was used to create a dynamic and interactive project presentation for showcasing system flow, features, and architecture

---

## 5. Coding & Documentation Standards
- **Code Standards:**  
  - Consistent naming conventions
  - Clean and readable code structure
  - Meaningful comments and modular design
- **Git Workflow:**  
  - Feature-based branches
  - Mandatory pull requests and code reviews before merging
- **Documentation:**  
  Each developer documented their implemented features, APIs, and system behavior.

---

## 6. Testing & Deployment
- **Testing:**  
  - Unit testing and integration testing were applied to ensure system stability
- **Deployment Process:**  
  - Development → Staging → Production
- **Responsible Member:**  
  Bara'a handled deployment supervision and final system validation before production release.

---

*This workflow, combined with the PERN Stack, Git/GitHub collaboration, and structured task management, ensured an efficient development process, clear communication, and a high-quality final product.*
